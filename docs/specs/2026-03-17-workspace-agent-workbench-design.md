# CCCLaw 工作区 Agent 工作台设计稿

**日期：** 2026-03-17

**目标：** 将 CCCLaw 的核心产品形态明确为“工作区驱动的 Agent 工作台”，并指导后续在现有原型基础上的界面改造。

## 1. 产品定位

CCCLaw 不是普通聊天产品，也不是传统后台系统。它的核心形态是：

- 用户进入平台后，在某个 `工作区` 中与 Agent 协作
- Agent 始终在当前工作区上下文中工作
- 对话是主交互方式
- 文件管理是工作区内的核心能力，而不是附属信息面板

产品主语应当是：

**“我进入一个工作区，在这里通过对话驱动 Agent 工作，并直接管理这个工作区里的文件。”**

而不是：

- “我在一个聊天产品里顺手选工作区”
- “我在一个后台系统里点开聊天模块”

## 2. 桌面端信息架构

桌面端采用稳定四栏结构：

1. **最左图标栏**
   - 承担全局导航
   - 保留较少的顶层入口，如工作台、任务、Provider、日志、设置
   - 视觉上作为整页锚点

2. **左侧工作区栏**
   - 只负责工作区切换
   - 不承载会话列表
   - 用户点击某个工作区后，直接切换到该工作区并恢复上次会话

3. **中间 Agent 对话区**
   - 整个页面的主舞台
   - 用户与 Agent 的持续协作在这里发生
   - 对话内容、流式回复、工具调用状态、输入框都集中在此

4. **右侧文件管理器**
   - 是完整工作区文件管理器，不是只读侧栏
   - 负责目录树浏览、文件/目录操作、文件内容查看

## 3. 核心布局规则

### 3.1 左侧工作区栏

左侧主栏只显示工作区相关内容：

- 工作区列表
- 当前工作区选中态
- 必要的工作区新增入口

不在这里展示：

- 会话列表
- 历史聊天
- 当前会话状态

原因是会话属于“当前聊天上下文”，应当归属于中间聊天区，而不是归属于工作区导航层。

### 3.2 中间聊天区头部

聊天区顶部承载当前工作语境，建议包含：

- 当前会话下拉
- 新建会话
- 工作区设置
- 必要时的轻量状态信息

交互规则：

- 切换工作区时，默认恢复该工作区的上次会话
- 用户想切换历史会话时，通过顶部会话下拉完成
- 用户想开始新任务时，通过顶部“新建会话”完成

因此，会话是“工作区内部的聊天对象”，但在 UI 上属于聊天区头部，而不是左侧导航。

### 3.3 右侧文件管理器

右侧区域定义为完整文件管理器，至少包含以下能力：

- 展开/收起目录
- 点击目录节点
- 点击文件节点
- 新建目录
- 新建文件
- 删除文件
- 删除目录
- 当前选中项高亮
- 文件内容查看

建议右侧内部再分为两层：

- 顶部操作栏：当前路径、`新建文件`、`新建目录`、更多操作
- 主体内容区：目录树 + 文件查看态

右侧虽然功能完整，但视觉权重仍低于中间聊天区。它是强操作副舞台，不应压过对话主轴。

## 4. 核心交互定义

### 4.1 工作区切换

- 用户点击左侧某个工作区
- 系统立即切换到该工作区
- 自动恢复这个工作区的上次会话
- 中间聊天区和右侧文件管理器同步切换到该工作区上下文

不要求用户切工作区后再手动选会话。

### 4.2 会话管理

会话相关操作统一放在聊天区顶部：

- 当前会话：以下拉形式显示
- 历史会话：在下拉面板中切换
- 新建会话：在当前工作区内创建空白会话

会话不进入左侧工作区栏，避免把“工作区导航”和“对话上下文”混成一层。

### 4.3 文件管理

右侧文件区支持完整操作链路：

- 点击目录：展开/折叠
- 点击文件：展示文件内容
- 新建目录：在当前目录下创建
- 新建文件：在当前目录下创建
- 删除文件/目录：在树节点上进行

文件查看建议采用右栏内部切换态，而不是让目录树和查看器同时过度拥挤。

## 5. 视觉风格

整体气质选择：

**现代工作台，轻量、精致、像 AI 原生产品。**

具体要求：

- 保留深色最左图标栏，作为视觉锚点
- 工作区栏、聊天区、文件区采用浅色层次
- 避免过重后台感
- 避免 IDE 式高密度压迫布局
- 对话区拥有最强的视觉呼吸感

风格目标是：

- 专业，但不僵硬
- 现代，但不花哨
- 明确是“Agent 工作台”，不是“普通聊天页”

## 6. 移动端设计原则

移动端不能直接复用桌面四栏并排布局，但必须保留相同的产品心智。

移动端主结构应为：

- 默认主视图：当前工作区的当前会话聊天区
- 工作区切换：通过左上角入口打开抽屉或全屏面板
- 文件管理器：通过右上角入口进入独立面板或覆盖层

移动端原则：

- 默认只聚焦聊天
- 工作区切换一步完成，并恢复上次会话
- 文件管理器仍需支持完整操作
- 文件查看在移动端应作为独立层级进入和返回
- 输入区、会话切换、工作区切换要适配单手操作

桌面与移动端的统一关系：

- 桌面端：工作区 / 对话 / 文件同时可见
- 移动端：对话为主，工作区和文件通过抽屉或覆盖层进入

## 7. 原型改造范围

后续原型修改应基于：

[`docs/prototypes/workspace-layout-prototype.html`](/Users/renlongyu/Desktop/ccclaw/docs/prototypes/workspace-layout-prototype.html)

修改方向如下：

1. 保留最左图标栏
2. 左侧第二栏只保留工作区列表
3. 移除左侧第二栏中的会话列表
4. 将会话切换改到中间聊天头部，采用“会话下拉 + 新建会话”
5. 保留工作区设置在聊天头部
6. 将右侧改造成完整文件管理器，而不是简单 tab 面板
7. 在右侧增加新建目录、新建文件、删除、目录展开、文件查看等原型交互
8. 为移动端补一套可用的抽屉式/覆盖式工作台结构

## 8. 设计结论

最终产品骨架如下：

- 最左：全局图标栏
- 左侧：工作区列表
- 中间：Agent 对话主区
- 中间顶部：会话下拉、新建会话、工作区设置
- 右侧：完整文件管理器

这是当前最符合 CCCLaw 产品定位的方案。
